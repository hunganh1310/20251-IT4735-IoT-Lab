# Contributing Guidelines

## For Students

This repository is for educational purposes as part of the IT4735 - IoT and Application course at HUST.

### Code Style

- Follow PEP 8 for Python code
- Use meaningful variable and function names
- Add comments to explain complex logic
- Keep functions small and focused

### Commit Messages

Use clear and descriptive commit messages:

```
Week X: Add [feature/fix] description

- Detail 1
- Detail 2
```

### Directory Structure

- Each week's lab should be in its own folder (e.g., `Week 5/`, `Week 7/`)
- Homework assignments go in `BTVN/` subfolder
- Keep data files (CSV, logs) organized

### Before Submitting

1. Test your code thoroughly
2. Remove any hardcoded credentials
3. Update documentation if needed
4. Ensure code runs without errors

## Questions?

Contact your course instructor or teaching assistant for help.
